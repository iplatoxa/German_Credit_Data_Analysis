## Описание проекта

Данный проект представляет собой комплексный анализ данных, направленный на выявление ключевых факторов, влияющих на кредитоспособность клиентов банка. На основе данных строится прогнозная модель для оценки риска дефолта по кредитам.

### Ключевые вопросы:
- Какие факторы наиболее сильно влияют на вероятность дефолта?
- Как кредитная история, финансовое положение и демография связаны с риском?
- Какие заемщики представляют наибольший риск для банка?

## Исходные данные

Датасет German Credit Data содержит следующую информацию о заемщиках:

| Поле | Описание |
|------|-----------|
| status | Текущий статус текущего счета |
| duration | Продолжительность кредита в месяцах |
| credit_history | Кредитная история |
| purpose | Цель кредита |
| amount | Сумма кредита |
| savings | Статус сберегательного счета |
| employment_duration | Трудоустройство в годах |
| default | Целевая переменная: Дефолт (1) / Нет дефолта (0) |

## Технологии и библиотеки

- **Python 3**
- **Анализ данных**: Pandas, NumPy
- **Визуализация**: Matplotlib, Seaborn
- **Статистика**: SciPy
- **Машинное обучение**: Scikit-learn

## Основные этапы анализа

1. **Предобработка данных**: Очистка, кодирование категориальных признаков, создание новых признаков
2. **Разведочный анализ (EDA)**: Визуализация распределений, анализ взаимосвязей, выявление паттернов
3. **Проверка гипотез**: Статистическая проверка влияния ключевых факторов на дефолт
4. **Машинное обучение**:
   - Подготовка данных (масштабирование, кодирование)
   - Обучение моделей (SVM, Random Forest, Gradient Boosting, Logistic Regression)
   - Оптимизация порогов классификации и оценка качества

## Ключевые выводы

### Результаты проверки гипотез

- **Кредитная история** — самый сильный фактор. Отсутствие истории рискованнее, чем плохая история (62.5% vs 34.0% дефолтов, p < 0.001)
- **Статус счета** — парадоксальный результат: отрицательный баланс рискованнее отсутствия счета (49.3% vs 11.7% дефолтов)
- **Сбережения** — минимальные сбережения значительно увеличивают риск (36.0% vs 20.9% дефолтов)
- **Параметры кредита** — более высокие суммы и длительные сроки связаны с повышенным риском

### Результаты моделирования

Лучшие результаты показала модель **SVM**:
- **AUC**: 0.688
- **Accuracy**: 72.5%
- **Recall дефолтов**: 25.0% (увеличивается до 75% при оптимизации порога)

После оптимизации порога классификации достигнут баланс:
- **Recall дефолтов**: 75%
- **Precision дефолтов**: 40%
- **Общая точность**: 65%

## Рекомендации для банка

1. **Основные факторы риска**: Модель должна в первую очередь учитывать кредитную историю и статус счета клиента
2. **Финансовая устойчивость**: Наличие сбережений и имущества значительно снижает риск дефолта
3. **Целевое кредитование**: Кредиты на бытовые нужды безопаснее, чем на престижные цели (новые авто, бизнес)
4. **Гибкая стратегия**: Использование различных порогов классификации для разных типов кредитных продуктов

## Бизнес-ценность

Проект демонстрирует возможность снижения кредитных рисков на 20-30% через внедрение системы скоринга на основе машинного обучения, обеспечивая баланс между минимизацией дефолтов и сохранением доходности кредитного портфеля.
